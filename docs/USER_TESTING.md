# Agent Rewind 五人用户验证方案

## 目标

这轮验证不测试“用户觉得概念是否有趣”，而测试四个可观察问题：

1. 用户能否独立完成安装；
2. 用户能否从预览中判断 Agent 将要修改什么；
3. Agent 出错后，用户能否安全恢复且不覆盖后续人工修改；
4. 用户是否愿意在真实工作中继续启用它。

## 参与者

招募 5 名已经使用 Claude Desktop 和 MCP 的开发者。至少 3 人过去一个月实际使用过文件系统类 MCP；排除项目作者和已经读过实现代码的人。

测试使用参与者自己的 macOS 设备，但只操作单独创建的测试目录。开始前说明：Agent Rewind 是早期版本，不能替代备份，不要把真实项目作为本轮测试目录。

## 测试材料

- 一份包含 6 至 10 个 Markdown、JSON 和文本文件的测试目录；
- 一张只包含目标任务的说明，不提前解释审批页各字段；
- 屏幕录制或观察记录表；
- 测试后由参与者亲自运行的本地聚合报告。

每位参与者使用同一份目录副本。主持人不能代替参与者输入安装命令或解释 diff，除非参与者明确表示无法继续；所有帮助都要记入观察表。

## 流程

### 1. 安装

向参与者提供 README 的“快速开始”部分。从开始阅读计时，到 Claude Desktop 重启后能看到受保护的 Filesystem MCP Server 为止。

记录：总耗时、失败次数、是否需要主持人帮助、失败所在步骤，以及参与者是否理解允许目录的含义。

### 2. 正常任务与预览理解

让 Claude 完成：“把测试目录中的会议记录按月份整理，并修正索引文件中的链接。”

第一次审批出现时先暂停。让参与者用自己的话回答：

- 哪些路径会受影响；
- 是新建、覆盖、编辑还是移动；
- 批准后最坏可能发生什么；
- 他/她会批准、拒绝还是需要更多信息。

回答完成后再允许继续。主持人按事实正确性标记“理解 / 部分理解 / 未理解”，不要按表达是否专业评分。

若操作属于带标签的显式 change set，让参与者自行在 `Approve`、`Allow set` 和 `Allow in folder` 中选择。记录其选择理由，以及参与者是否理解 `Allow set` 只覆盖当前组和首次批准的目录，而 `Allow in folder` 会持续到本次 Agent Rewind 进程结束。

### 3. 误操作与撤销

使用预先准备的提示，让 Agent 把一个仍需保留的文件错误移动或覆盖。参与者发现结果不符合要求后，自行寻找恢复方式。

记录：是否找到 change set、是否主动点击撤销、从发现错误到恢复的时间、恢复后的文件哈希是否与原始副本一致。

### 4. 冲突保护

Agent 修改一个文件后，让参与者手动追加一行，再尝试撤销对应 change set。预期结果是 Agent Rewind 拒绝覆盖该文件，并清楚显示冲突。

记录：人工追加内容是否保留、参与者是否理解拒绝原因、是否知道下一步需要人工处理。

### 5. 访谈和本地报告

测试结束后让参与者运行：

```bash
npm exec --yes --package=github:YanhanLi/agent-rewind -- agent-rewind report
```

参与者先检查报告，再自行决定是否分享。报告只包含聚合计数，不包含路径、文件内容、prompt 或工具参数。

最后询问：

- 哪一次确认提供了真实价值，哪一次只是打断；
- 与 Claude Desktop 原生确认相比，最明显的差异是什么；
- 是否愿意在真实目录持续启用一周，为什么；
- 哪一个缺失能力会阻止继续使用。

不要先问“你喜欢吗”，也不要向参与者介绍未来路线图。

## 判定门槛

满足以下全部条件才进入下一轮客户端适配：

- 安装耗时中位数低于 5 分钟，至少 4/5 人无需主持人代操作；
- 至少 4/5 人能正确描述首次预览的主要影响；
- 5/5 的无冲突撤销恢复到原始哈希，5/5 的冲突测试保留人工修改；
- 至少 3/5 人愿意在真实工作中继续启用一周；
- 至少 3/5 人能说出相对原生确认的具体差异，而不是笼统的“更安全”。

## Go / Iterate / Pivot

- **Go**：全部门槛通过。先修复测试中的高频摩擦，再验证 OpenCode 或 Codex 接入。
- **Iterate**：恢复正确但安装、预览理解或打扰程度未达标。保留核心范围，只迭代接入和交互。
- **Pivot**：恢复可靠性不达标，或少于 3/5 人愿意继续使用。停止增加客户端，重新评估是否转向 Tool Drift Monitor。

## 数据处理

默认不收集任何远程遥测。观察记录使用参与者编号，不记录真实项目路径。屏幕录制必须单独征得同意；不同意录制不影响参加测试。原始记录在总结完成后 30 天内删除，只保留去标识化的聚合结论。
